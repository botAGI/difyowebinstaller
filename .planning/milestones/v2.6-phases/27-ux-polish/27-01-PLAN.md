# _stream_gpu_model_logs — stream docker logs -f for a GPU service until healthy or timeout.
# Usage: _stream_gpu_model_logs "agmind-vllm" "vLLM" 600
# In TTY: streams raw docker logs -f to stdout.
# In non-TTY: polls docker logs --tail=1 every 10s, prints status line.

    local label="$2"

Implementation details:
- Check `[ -t 1 ]` for TTY detection
- **TTY path:** Run `docker logs -f --since=0s "$container" 2>&1 &` as background process, store PID. Poll container health via `docker inspect --format='{{.State.Health.Status}}' "$container"` every 5 seconds. Also track inactivity: if no new log output for 60 seconds, consider stalled. On healthy or timeout, `kill $logs_pid 2>/dev/null; wait $logs_pid 2>/dev/null || true`.
- **Non-TTY path:** Loop every 10 seconds, run `docker logs --tail=1 --no-log-prefix "$container" 2>/dev/null | tr -d '\r'` and print a single status line: `log_info "${label}: ${last_line_summary}"` (truncate to 80 chars). Check health same way.
- Return 0 if container health becomes "healthy", return 1 if timeout reached.
- Use `local elapsed=0` counter incremented by sleep interval.
