Modify `_pull_with_progress()` in `lib/compose.sh` to detect and report missing images after the background pull completes. Changes:

1. **Capture pull exit code** (line 76 area): Replace `wait "$pull_pid" || true` with:

2. **Add `_validate_pulled_images()` helper function** right after `_print_pull_status()` (after line 105). This function:
   - Takes the images array by nameref and total count
   - Iterates over all images, checks each with `docker image inspect`
   - For each MISSING image: prints `log_error "Образ не найден: ${img}. Проверьте тег в versions.env"`
   - Returns the count of missing images

   ```bash
   _validate_pulled_images() {
       local -n _vimgs=$1
       local total="$2"
       local missing=0

       for img in "${_vimgs[@]}"; do
           if ! docker image inspect "$img" >/dev/null 2>&1; then
               log_error "Образ не найден: ${img}. Проверьте тег в versions.env"
               missing=$((missing + 1))
           fi
       done

       return "$missing"
   }
   ```

3. **Replace the final ready-count block** (lines 78-84) with:
   ```bash
   # Clear progress line, print final result
   printf "\r%-80s\r" ""
   local ready=0
   for img in "${images[@]}"; do
       docker image inspect "$img" >/dev/null 2>&1 && ready=$((ready + 1))
   done

   if [[ $ready -eq $total ]]; then
       log_success "Images ready: ${ready}/${total}"
   else
       log_warn "Images ready: ${ready}/${total}"
       _validate_pulled_images images "$total" || true
   fi

   # Return non-zero if any images are missing (caller decides severity)
   [[ $ready -lt $total ]] && return 1
   return 0
   ```

2. **Modify `pull_model()` function** (lines 42-60):

   - Use `docker exec -t` (allocate pseudo-TTY) instead of bare `docker exec` for progress display
   - Add fallback: if `-t` fails (no TTY available, e.g. piped output), retry without `-t`

   Replace the function body with:
   ```bash
   pull_model() {
       local model="$1"
       local label="${2:-$model}"

       # Show approximate size if known
       local size="${MODEL_SIZES[$model]:-}"
       if [[ -n "$size" ]]; then
           log_info "Downloading model: ${label} (~${size})..."
       else
           log_info "Downloading model: ${label}..."
       fi

       # Validate model name
       if [[ ! "$model" =~ ^[a-zA-Z0-9.:/_-]+$ ]]; then
           log_error "Invalid model name: ${model}"
           return 1
       fi

       # Try with TTY for progress display, fallback to non-TTY
       if docker exec -t agmind-ollama ollama pull "$model"; then
           log_success "Model ${label} downloaded"
       elif docker exec agmind-ollama ollama pull "$model"; then
           log_success "Model ${label} downloaded (no progress display)"
       else
           log_error "Failed to download model ${label}"
           return 1
       fi
   }
   ```

   Key points:
   - `docker exec -t` allocates a pseudo-TTY so Ollama shows its progress bar (layers, percentages)
   - If `-t` fails (non-interactive context), falls back to `docker exec` without TTY
   - Size display uses the hardcoded lookup — unknown models skip size
