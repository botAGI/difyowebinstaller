<objective>
Fix two critical bugs discovered after v2.3 release:

1. BFIX-41: NON_INTERACTIVE VRAM guard bypass -- when NON_INTERACTIVE=true, _wizard_llm_model() returns early or assigns default VLLM_MODEL without running VRAM validation. Models exceeding GPU memory silently proceed, causing OOM at runtime.

2. BFIX-42: Resume diagnostics gap -- resuming install from phase >= 2 skips phase_diagnostics, leaving DETECTED_OS/DETECTED_GPU_VRAM unset. Later phases using these variables crash with "unbound variable".

Purpose: Prevent silent OOM deployments and resume crashes -- the two most impactful bugs in v2.3.
Output: Patched lib/wizard.sh and install.sh.
</objective>
