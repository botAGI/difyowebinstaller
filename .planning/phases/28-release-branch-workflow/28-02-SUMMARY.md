# Phase 28 Plan 02: Release Branch Workflow Summary

Git-based update mechanism: `agmind update` pulls scripts from `release` branch via `git pull --ff-only`, `--main` enables one-time dev fetch, `.gitignore` covers all installer-generated files to prevent merge conflicts.

## What Was Built

### .gitignore expansion (Task 1)

Added a new section covering all files generated by `install.sh` at runtime:

- `credentials.txt`, `RELEASE`, `docker/`, `logs/`, `.rollback/`
- `.dify_initialized`, `.admin_password`, `.checkpoint_*`
- `*.bak`, `*.tmp`

This ensures `git pull` during `agmind update` never encounters merge conflicts from runtime state (Pitfall M-03).

### Git-based update mechanism (Task 2)

**`scripts/update.sh` changes:**

- Added `UPDATE_BRANCH="${UPDATE_BRANCH:-release}"` and `SCRIPTS_ONLY="${SCRIPTS_ONLY:-false}"` defaults
- Added `--main` flag: sets `UPDATE_BRANCH=main` for one-time dev fetch; reverts to `release` on next run
- Added `--scripts-only` flag: exits after `update_scripts()` without triggering docker pull
- New `update_scripts()` function:
  - Validates INSTALL_DIR is a git repo
  - Stashes local changes before pull (Pitfall C-01)
  - `git fetch origin $branch` + `git checkout $branch` + `git pull --ff-only origin $branch`
  - Warns on `--main` usage: "One-time pull from 'main' complete. Next 'agmind update' will use 'release'."
- Modified `fetch_release_info()`:
  - GitHub Releases API fetch now non-fatal: 403/429 logs warning, continues with branch-fetched versions.env (Pitfall N-04)
  - `versions.env` now fetched from `raw.githubusercontent.com/botAGI/AGmind/${UPDATE_BRANCH}/templates/versions.env`
  - Removed `RELEASE_VERSIONS_URL` asset-search logic and its "no versions.env asset" fatal error
- Modified `main()` flow: calls `update_scripts()` before `fetch_release_info()`; exits early if `SCRIPTS_ONLY=true`

**`scripts/agmind.sh` changes:**

- Added `--scripts-only` to `cmd_help()` update section
- `--main` intentionally NOT added to help text (dev-only, per plan decision)
- Flag passthrough via `exec "${SCRIPTS_DIR}/update.sh" "$@"` already covers both new flags

**`README.md` changes:**

- Quick Start updated: primary method is `curl -fsSL .../release/bootstrap.sh | sudo bash`
- Alternative: `git clone -b release` (explicit branch)
- CLI table updated: `agmind update` describes release branch pull; `agmind update --scripts-only` added
