**1. Telegram HTML escape in update.sh:**

Add `_escape_html()` function BEFORE `send_notification()` (around line 159):

```bash
# Escape HTML special characters for Telegram parse_mode=HTML
_escape_html() {
    local text="$1"
    # Order matters: & first to avoid double-escaping
    text="${text//&/&amp;}"
    text="${text//</&lt;}"
    text="${text//>/&gt;}"
    echo "$text"
}
```

Modify `send_notification()` to escape the message at the top of the function, right after `local message="$1"`:

```bash
send_notification() {
    local message="$1"
    message="$(_escape_html "$message")"
    [[ ! -f "$ENV_FILE" ]] && return 0
    # ... rest unchanged ...
```

Note: The webhook path does JSON escaping already (sed on line 188) -- HTML escape is harmless there since JSON doesn't care about `&amp;`, but the function is called before the case switch so it applies to both paths. This is acceptable since webhook consumers handle HTML entities fine.
