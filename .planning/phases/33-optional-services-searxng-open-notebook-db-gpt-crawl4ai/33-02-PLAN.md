```bash
        sed "s|__SEARXNG_SECRET_KEY__|${_SEARXNG_SECRET_KEY}|g" \
            "${INSTALL_DIR}/templates/searxng-settings.yml" \
            > "${INSTALL_DIR}/docker/searxng-settings.yml"

```

Wait — actually the docker-compose.yml mounts `../templates/searxng-settings.yml` directly. So instead, we should do the sed replacement IN-PLACE on the templates file after copying, OR change the mount path. Better approach: change docker-compose.yml volume mount to `./searxng-settings.yml:/etc/searxng/settings.yml:ro` (relative to docker/ dir) and generate the processed file into `docker/searxng-settings.yml`. This is consistent with how `litellm-config.yaml` is generated into `docker/`.

So the actual action for config.sh is: generate `${INSTALL_DIR}/docker/searxng-settings.yml` from the template, replacing `__SEARXNG_SECRET_KEY__`.

And in Plan 01's docker-compose.yml, the volume mount should be `./searxng-settings.yml:/etc/searxng/settings.yml:ro` (not `../templates/...`). If Plan 01 already used `../templates/...`, update it to `./searxng-settings.yml:/etc/searxng/settings.yml:ro` in this task as well.
