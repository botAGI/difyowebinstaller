**Fix 1 — install.sh line ~181:**
In the `_copy_runtime_files()` function, add `redis-lock-cleanup.sh` to the `scripts` array. Insert it after `generate-manifest.sh` (or anywhere in the array). The resulting line should be:

```bash
local scripts=(backup.sh restore.sh uninstall.sh update.sh agmind.sh health-gen.sh rotate_secrets.sh dr-drill.sh build-offline-bundle.sh generate-manifest.sh redis-lock-cleanup.sh)
```

This ensures `scripts/redis-lock-cleanup.sh` from the repo is copied to `${INSTALL_DIR}/scripts/redis-lock-cleanup.sh` BEFORE `compose_up()` runs (phase 4 config runs before phase 5 start). Without this, Docker sees a bind mount target that does not exist and creates a directory, which causes the lock cleaner to silently fail.

**Fix 2 — lib/compose.sh compose_up() function:**
After the existing line `local profiles="$COMPOSE_PROFILE_STRING"` (around line 113), add code to persist COMPOSE_PROFILES to the .env file so that systemd can read it on reboot:

```bash
    # Persist COMPOSE_PROFILES to .env for systemd reboot support (BUG-V3-029)
    local env_file="${INSTALL_DIR}/docker/.env"
    if [[ -f "$env_file" ]]; then
        # Remove any existing COMPOSE_PROFILES line first
        sed -i '/^COMPOSE_PROFILES=/d' "$env_file"
        echo "COMPOSE_PROFILES=${profiles}" >> "$env_file"
    fi
```

Insert this block AFTER `local profiles="$COMPOSE_PROFILE_STRING"` and BEFORE `_cleanup_stale_containers`. The sed -i removes any previous COMPOSE_PROFILES line to avoid duplicates on re-runs, then appends the current value.

The full [Service] section should become:

```ini
[Service]
Type=oneshot
RemainAfterExit=yes
WorkingDirectory=__INSTALL_DIR__/docker
EnvironmentFile=-__INSTALL_DIR__/docker/.env

ExecStartPre=/bin/bash -c 'for i in $(seq 1 30); do docker info >/dev/null 2>&1 && exit 0; sleep 1; done; exit 1'
```
