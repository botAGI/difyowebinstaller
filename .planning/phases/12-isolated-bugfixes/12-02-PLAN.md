The bug: `check_component()` stores `latest` (the raw GitHub tag_name like `v1.36.6`) in the UPDATES array at line 223. For components whose Docker Hub images use tags WITHOUT v-prefix (Weaviate, Postgres, Redis, Grafana, and others), the report shows incorrect version like `v1.36.6` instead of `1.36.6`.

Fix approach: Add an associative array of components that need v-prefix stripped from the `latest` value before writing to UPDATES, and apply the strip in `check_component()`.

**Step 1:** After the `UPDATES=()` line (line 54), add a new associative array:

```bash
# Components whose Docker images do NOT use v-prefix in tags.
# GitHub releases use v-prefix (v1.36.6) but Docker Hub has bare (1.36.6).
declare -A NO_V_PREFIX=(
    [Weaviate]=1
    [PostgreSQL]=1
    [Redis]=1
    [Grafana]=1
    [Nginx]=1
    [Prometheus]=1
    [Alertmanager]=1
    [Loki]=1
    [Promtail]=1
    [Node Exporter]=1
    [cAdvisor]=1
)
```

Note: All hub-sourced components (Postgres, Redis, Nginx, Squid, Portainer) already return bare tags from Docker Hub API (no v-prefix), so those won't be affected. The main targets are GitHub-sourced components that use v-prefix tags but whose Docker images expect bare versions: Weaviate, Grafana, Prometheus, Alertmanager, Loki, Promtail, Node Exporter, cAdvisor.

**Step 2:** In `check_component()`, right before the `UPDATES+=` line (line 223), add v-prefix stripping:

```bash
    if is_newer "$current" "$latest"; then
        local change
        change=$(classify_change "$current" "$latest")
        # Strip v-prefix for components whose Docker images don't use it
        local report_latest="$latest"
        if [[ -n "${NO_V_PREFIX[$name]+x}" ]]; then
            report_latest="${latest#v}"
        fi
        UPDATES+=("${name}|${current}|${report_latest}|${change}")
        echo "  UPDATE: ${name} ${current} -> ${report_latest} (${change})" >&2
    else
```

This changes two lines: the UPDATES+= line now uses `report_latest` instead of `latest`, and the echo line also uses `report_latest`.

Do NOT modify normalize_version(), classify_change(), is_newer(), or generate_report() — the fix is only in check_component() and the new array declaration. The comparison logic (is_newer, classify_change) already uses normalize_version() which strips v internally, so comparison is unaffected.
