Create a GitHub Release using the `gh` CLI:

1. First verify `gh` is authenticated:
   ```bash
   gh auth status
   ```

2. Create the release with tag, title, notes, and asset:
   ```bash
   gh release create v2.1.0 \
     --repo botAGI/difyowebinstaller \
     --target main \
     --title "v2.1.0 — Initial Stable Release" \
     --notes "$(cat <<'NOTES'
   ## AGmind Installer v2.1.0 — Initial Stable Release

   First official release of the AGmind all-in-one AI stack installer.

   ### What's included

   - **One-command install:** `sudo bash install.sh` deploys the full stack
   - **4 deployment profiles:** LAN, VPN, VPS, Offline
   - **Provider choice:** Ollama, vLLM, or external LLM/embedding provider
   - **24-34 containers** managed via Docker Compose profiles
   - **Security hardened:** Authelia 2FA, nginx rate limiting, SSRF sandbox, credential isolation
   - **Day-2 operations:** `agmind status`, `agmind doctor`, `agmind update`, backup/restore
   - **Monitoring stack:** Grafana, Prometheus, Loki, Alertmanager

   ### Tested on

   - Ubuntu 24.04 LTS
   - NVIDIA RTX 5070 Ti (535+ driver)
   - 24/24 containers healthy after fresh install
   - Resume from checkpoint verified
   - Backup/restore cycle verified

   ### Components (versions.env attached)

   All component versions are pinned in the attached `versions.env` file.
   See `COMPONENTS.md` in the repo for dependency group documentation.

   ### Upgrade path

   This is the baseline release. Future releases will include `versions.env` as an asset
   for the bundle update system (`agmind update`).
   NOTES
   )" \
     templates/versions.env
   ```

3. Verify the release was created:
   ```bash
   gh release view v2.1.0 --repo botAGI/difyowebinstaller
   ```

4. Verify versions.env is listed as an asset:
   ```bash
   gh release view v2.1.0 --repo botAGI/difyowebinstaller --json assets --jq '.assets[].name'
   ```

If `gh auth status` fails, create a checkpoint for the user to authenticate.
