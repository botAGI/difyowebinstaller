Modify `templates/authelia/configuration.yml.template` access_control section.

The current config is:
```yaml
access_control:
  default_policy: one_factor
  rules:
    - domain: '__DOMAIN__'
      resources:
        - '^/console.*$'
      policy: two_factor
```

Replace the entire `access_control` block with:
```yaml
access_control:
  default_policy: one_factor
  rules:
    # API routes bypass Authelia — Dify handles its own API key auth
    # Compensated by nginx rate limiting (10r/s on /api, /v1, /files)
    - domain: '__DOMAIN__'
      resources:
        - '^/api/.*$'
        - '^/v1/.*$'
        - '^/files/.*$'
      policy: bypass
    # Console routes require 2FA (human login)
    - domain: '__DOMAIN__'
      resources:
        - '^/console.*$'
      policy: two_factor
```

The `bypass` rule for API routes must come BEFORE the `two_factor` rule because Authelia evaluates rules top-to-bottom, first match wins. Without the explicit bypass, the `default_policy: one_factor` would apply to API routes, which would still break programmatic access. The explicit bypass makes it auditable.

Do NOT change any other section of the file (server, log, jwt_secret, totp, authentication_backend, session, storage, notifier).
