1. **templates/nginx.conf.template** -- Add LiteLLM upstream block after the `upstream openwebui` block (around line 63). Use the docker-compose service name (not container_name), matching existing patterns (`server api:5001`, `server open-webui:8080`):
```nginx
    # Upstream: LiteLLM AI Gateway
    upstream litellm {
        server litellm:4000;
    }
```

Add a `/litellm/` location block inside the HTTP server (port 80) block, BEFORE the `# Block old-style hex token URLs` section (before the `location ~ "^/[a-f0-9]{24,}/"` line). Place it after the Open WebUI `/` location block:

```nginx
        # =============================================
        # LiteLLM UI — AI Gateway admin dashboard
        # =============================================
        location /litellm/ {
            proxy_pass http://litellm/ui/;
            proxy_http_version 1.1;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
        }
```

Also add the same location block inside the `#__TLS__` HTTPS server block (before the `# Block old-style hex token URLs` line in the TLS section):
```nginx
    #__TLS__    # LiteLLM UI
    #__TLS__    location /litellm/ {
    #__TLS__        proxy_pass http://litellm/ui/;
    #__TLS__        proxy_http_version 1.1;
    #__TLS__        proxy_set_header Host $host;
    #__TLS__        proxy_set_header X-Real-IP $remote_addr;
    #__TLS__        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    #__TLS__        proxy_set_header X-Forwarded-Proto $scheme;
    #__TLS__    }
```

NOTE: proxy_pass uses `http://litellm/ui/` -- LiteLLM serves its admin UI at `/ui/` path. The trailing slash in both `location /litellm/` and `proxy_pass http://litellm/ui/` ensures proper path rewriting.

    if docker ps --format '{{.Names}}' 2>/dev/null | grep -q 'agmind-litellm'; then
        if docker exec agmind-litellm curl -sf --max-time 5 http://localhost:4000/health >/dev/null 2>&1; then
            _check OK "LiteLLM Gateway" "healthy (port 4000)"

            _check WARN "LiteLLM Gateway" "Container running but health check failed" "docker compose restart agmind-litellm"
        fi
    fi
